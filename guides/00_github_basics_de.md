# GitHub-Basics (5 Minuten): So klappt die Mitarbeit ohne Vorkenntnisse

Du musst kein „Git-Profi“ sein, um bei VergabeLab Community mitzuwirken. Für die meisten Beiträge reicht ein einfacher Beitrag im Forum (Discussions) oder ein Issue.

## A) Der einfachste Weg: Eine Discussion (für Ideen & Fragen)
Wenn du eine Idee für einen neuen Baustein hast, Feedback geben möchtest oder eine rechtliche Frage hast:
1. Öffne die Discussions: https://github.com/vergabelab/vergabelab-community/discussions
2. Klicke auf **New discussion**.
3. Wähle eine passende Kategorie (z. B. "Ideen" oder "Q&A").
4. Schreib deinen Beitrag einfach wie in einem normalen Forum.

## B) Ein Issue erstellen (für konkrete Fehler & Aufgaben)
Wenn du einen Tippfehler, einen inhaltlichen Fehler (z. B. eine Halluzination der KI bei einem bestimmten Prompt) gefunden hast:
1. Öffne die Issue-Seite: https://github.com/vergabelab/vergabelab-community/issues
2. Klicke auf **New issue**.
3. Wähle (falls vorhanden) eine Vorlage aus.
4. Beschreibe kurz:
   - **Worum geht’s?**
   - **Welche Datei betrifft es?** (z. B. `prompts/markterkundung/...`)
   - **Was ist dein Lösungsvorschlag?**

## C) Dateien lesen und kopieren (oder Download)
- Klicke im Repository auf eine Datei (z. B. unter `prompts/`).
- Nutze den Button **Copy raw file** (das kleine Kopieren-Symbol oben rechts an der Datei) oder markiere den Text und kopiere ihn ganz normal.

## D) Pull Request (optional für Fortgeschrittene)
Wenn du Änderungen direkt im Textartefakt vorschlagen willst:
- Du kannst eine Datei direkt hier im Browser bearbeiten (Klick auf das Stiftsymbol „Edit“).
- Wenn du deine Änderungen speicherst, erstellt GitHub daraus automatisch einen konkreten Änderungsvorschlag (Pull Request).
*Das ist komplett optional – ein Issue oder eine Discussion reicht völlig aus!*
> **Hinweis:** Für Pull Requests ist ein DCO-Signoff erforderlich. Wenn du über die GitHub-Weboberfläche arbeitest, ergänzt GitHub den Signoff automatisch (da dies für das VergabeLab-Repository aktiviert ist). Wenn du deine private E-Mail-Adresse nicht veröffentlichen möchtest, aktiviere in GitHub: ``` Settings → Emails → Keep my email addresses private```. GitHub zeigt dir dort eine noreply-Adresse an. Diese kannst du für Signoffs und Commits verwenden.

## ⚠️ Wichtiger Hinweis für Vergabestellen und sonstige Nutzer
Bitte poste **keine personenbezogenen Daten**, **keine Details laufender Vergaben** und **keine Betriebs- und Geschäftsgeheimnisse**. Beispiele bitte immer anonymisieren (z. B. „Bieter A/B“, „Projekt X“).
